## 电子商务网站首页开发

技术栈：CSS3  HTML5  原生JavaScript

---
总结：
```js
1：首先要对网站的布局有一个清晰地认识，明白这个网站有哪几个功能模块组成，哪些模块是可以重用的，哪些地方是不可以重用的
（对于可重用的部分，我们会使用 class 来书写样式代码）。

2：然后搭建整个网站的大体框架，比如说header、content/container、footer。

3：最后开始去处理每一个模块，在制作每一个模块的时候遵循先内容后样式，在书写内容和样式的时候要记得语义化，同时要明白哪
些地方用的是 img、哪些地方用的是背景图，哪些地方用的是平铺。

4：等整个网站的内容和样式全部完成以后再去开始 JS 代码的书写。
```

---
需要注意的一些细节：
```js
网页中经常替换掉的文件用单独的一个文件夹来存放，便于后期的维护。

 为了便于维护，最好别不同功能区域的图片单独专门放一个文件夹。

用 a 链接做页面索引的时候，a 的高是可以定死的，但是宽最好是用 padding 来做，具有扩展性。

在制作图片滚动列表时，图片是在一个固定的可视区范围内滚动的。

网站从头到尾做完之后，去在各个浏览器里看一看效果，看一下兼容性问题。

背景色，可以用div来做，然后里面再包一个div来放内容，这样做的好处就是如果你要替换其中的内容的时候很方便，但是这样会写很多
代码。你也可以竖着截取一个像素的宽度，然后平铺开来。(切像素的多少，直接与性能相关，因为你切的像素越宽，它走的步数就越少)，
注意存储为 web 使用格式。存储为何种格式，有一个原则，就是在图片质量不是特别损失的情况下，用图片所占内存最小的格式。这两个
方法其实都是可以的，主要是看你具体的业务需求。

为了更好的语义化，在 img 元素中最好把 alt 和 title 属性都写上，为了更好的 seo。
```

---
兼容性问题：
```js
如果一个布局和样式是合理的，良好的，基本上所碰到的兼容性的问题是屈指可数的几个。

在本次的开发中，兼容性问题主要出现在 IE6，其中的几个问题是：png 图片的兼容性问题（ie6 是不认 png 图片的），这个问题可
以通过以下的代码解决：

<!--[if IE 6]>
<script src="js/DD_belatedPNG_0.0.8a.js"></script>
<script>
DD_belatedPNG.fix('*');
</script>
<![endif]-->

在 ie6 下 line-height 这个属性是不可以针对图片进行设置的，也就是说这个行高没有办法让这个图片居中。我们可以把图片变成行
内元素，然后再进行设置。让图片变成块的方法有很多，浮动可以，display 也可以。

解决 ie6 下的问题有一个原则：尽量不要用 hack，它向后的扩展是不好的。

在 ie6 的文本框中出现的文本是不会居中的，所以你这个时候给它一个行高就可以了。

下 ie6 下需要给 a 链接设置 cursor:pointer;。

在 ie6 下使用 png 图片，ie6 总是会让其多出来一到两个像素，这个时候我们可以换其他格式的图片使用来解决这个问题。

在 ie6 下对于有 hover 或者没有 hover 的状态都需要设置 text-decoration:none；。

```

如何书写你的 JS ？
```js
如何在一个网站或者一个页面去书写你的js代码：这个时候你需要对你所写的js有一个整体的整合，你只要遵循以下两点就可以了：

- js 的分层处理（功能性的）：

底层库（比如说jQuery） 

组件库（所谓组件，就是说当你找到底层之后，你的页面里面还有有一些基本的功能，包括像拖拽，弹窗之类的，这些不仅可以在一个项
目中可以用，在另一个项目中也是可以用的，这些组件没有一个必然的练习，但是把它们组合到一起，就可以形成一个你想要的效果。）  

具体的应用

- js的规划（管理） 避免全局的变量和方法（可以采用命名空间，闭包，面向对象把这些代码进行更合理的规划，不让她出现问题，从
而方便管理。也可以通过现在比较流行的模块化来做seaJs requireJs ）
```
